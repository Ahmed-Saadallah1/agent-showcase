About this agent

What it does
Auto-detects whether an input name is in Arabic or English script, then transliterates it into the other script based on pronunciation. Supports both single names and batches (one name per line). It is built to never translate meaning -- for example, a name containing the Arabic word for "rose" is kept as a phonetic name, not translated into "rose."
Model used
deepseek/deepseek-v4-flash, via OpenRouter
About this model
DeepSeek V4 Flash is an efficiency-focused model built for fast, responsive output while maintaining strong reasoning and language performance. Note: unlike this showcase's other agents, this model is a paid OpenRouter model rather than a free-tier one.
How it works
A LangChain tool-calling agent detects the input script in code (not by the model), then asks the model to transliterate phonetically, with explicit instructions preserving pronunciation and formatting rules to keep the output clean. Batches are processed one name at a time.
